# 未找到与重复对象可视化设计

**项目：** 傑出品 Navisworks 查找插件

**日期：** 2026-07-16

**状态：** 已确认，等待实施计划

**目标版本：** Navisworks Manage 2023 / .NET Framework 4.8

## 1. 背景

当前插件会对每条输入条件执行搜索，并在结果页显示查询值和匹配数量。用户可以知道某条条件是否为零命中，但无法快速回答以下问题：

- 未找到的是哪一条完整条件；
- 条件是正常执行后零命中，还是根本没有成功构造；
- 一条条件是否命中了多个具有相同属性值的模型对象；
- 结果异常时是否仍然允许执行隐藏未选中；
- 如何从结果快速返回原始条件或定位重复对象。

项目的业务约束已经确认：每条输入查询条件在当前选定模型范围内必须恰好匹配 1 个对象。

因此，本功能不是单纯增加颜色，而是把现有“是否存在”判断升级为“结果数量是否符合唯一性约束”的验证流程。

## 2. 目标

本设计需要实现以下结果：

1. 每条条件都有明确、互斥且可解释的结果状态。
2. 用户可以在结果页一眼看出未找到、重复和条件异常。
3. 结果必须包含完整条件信息，能够区分相同查询值对应的不同条件。
4. 重复结果能够关联并选中实际命中的多个模型对象。
5. 任何不满足唯一性约束的结果都必须阻止隐藏未选中。
6. 条件发生变化后，旧搜索结果必须立即失效。
7. 不改变当前搜索范围、多条件并集、STR 保护和对象去重规则。

## 3. 非目标

本次不包含以下能力：

- 不在整个模型中自动复查当前范围之外的对象；
- 不在三维视图中生成不存在对象的虚拟几何体；
- 不支持每条条件配置不同的期望数量；
- 不把不同条件命中同一个对象视为模型值重复；
- 不改变 XML 条件之间现有的并集语义；
- 不重新设计搜索条件编辑窗口或插件其他选项页；
- 不引入批量 XML 处理。

## 4. 核心定义

### 4.1 搜索范围

所有状态只针对本次搜索开始时快照的当前选定模型范围。一个对象即使存在于范围之外，也不属于本次搜索结果。

界面和导出内容必须使用“当前选定范围内”这一表述，避免被理解为整个模型不存在。

### 4.2 模型值重复

模型值重复是指：同一条查询条件在当前选定范围内命中了两个或更多不同的 `ModelItem`。

以下情况不属于本设计中的模型值重复：

- 输入列表中出现两条内容相同的条件；
- 两条不同条件命中了同一个模型对象；
- 同一个对象同时具有多个相同显示文本的属性。

### 4.3 唯一性约束

所有输入条件都采用相同约束：必须恰好匹配 1 个对象。

匹配方式 `equals` 和 `contains` 只决定如何比较属性值，不改变唯一性约束。即使使用 `contains`，本项目仍要求最终只命中 1 个对象。

## 5. 状态模型

每条条件只能处于以下四种状态之一：

| 条件执行情况 | 匹配数量 | 状态 | 说明 | 允许隐藏 |
|---|---:|---|---|---|
| 正常完成 | 0 | 未找到 | 当前选定范围内没有匹配对象 | 否 |
| 正常完成 | 1 | 已找到 | 唯一匹配，符合约束 | 是 |
| 正常完成 | 大于等于 2 | 重复 | 当前选定范围内存在多个相同值对象 | 否 |
| 未能构造或验证 | 不适用 | 条件异常 | 条件没有真正执行 | 否 |

### 5.1 条件异常范围

条件异常至少包括：

- 必填属性信息为空；
- 缺少分类且无法从模型采样发现分类；
- 匹配方式不是插件明确支持的类型；
- 其他在调用 Search API 之前即可明确判断的条件错误。

Navisworks Search API 自身抛出异常不归类为某一行条件异常。本次搜索应整体中止并显示错误，避免把部分结果伪装成完整结果。

### 5.2 状态优先级

状态按以下顺序判定：

1. 先验证条件是否可以执行；失败则为条件异常。
2. 条件正常执行后读取匹配数量。
3. 根据 0、1、大于等于 2 判定未找到、已找到或重复。

## 6. 结果数据设计

每条结果必须保留本次搜索的条件快照，而不能只保存查询值。

结果信息包括：

- 原始条件序号；
- 分类内部名和显示名；
- 属性内部名和显示名；
- 匹配方式；
- 查询值；
- 结果状态；
- 状态说明；
- 匹配数量；
- 实际命中的模型对象集合。

原始条件序号用于在结果与搜索条件表格之间建立稳定关联。完整条件快照用于区分查询值相同但分类或属性不同的条件。

结果状态应由匹配层产生，界面只负责展示，不在 UI 中重复推断业务状态。

## 7. 架构影响

### 7.1 搜索结果模型

扩展结果模型，使其承载完整条件快照、状态和说明。匹配对象集合继续作为对象选择、重复定位和选择集创建的数据来源。

### 7.2 匹配引擎

匹配引擎负责：

- 为每条条件建立稳定序号；
- 验证条件是否可执行；
- 区分无法构造条件与正常零命中；
- 根据匹配数量生成四态结果；
- 保持每条条件独立执行的现有语义。

### 7.3 搜索编排

主搜索流程继续负责：

- 快照当前选择范围；
- 识别唯一模型前缀；
- 执行条件搜索；
- 汇总结果；
- 选中匹配对象；
- 处理 STR 保护；
- 应用隐藏门禁；
- 保存结果和日志。

新增职责仅限于根据结果状态决定是否允许进入隐藏确认。

### 7.4 结果页

结果页从文本列表升级为结构化表格。结果页不重新计算状态，也不直接改变搜索条件内容。

### 7.5 导出与日志

导出和诊断日志读取结果模型中的统一状态，避免界面、日志和 CSV 对同一结果产生不同解释。

## 8. 界面设计

采用已经确认的“方案 A：结果页结构化表格”。

### 8.1 顶部汇总

顶部显示：

- 条件总数；
- 已找到数量；
- 未找到数量；
- 重复数量；
- 条件异常数量；
- 当前搜索范围或模型前缀；
- 去重后的总匹配对象数。

统计区域使用低饱和背景和状态色边线，不使用大面积高饱和颜色。

### 8.2 筛选

提供以下筛选：

- 全部；
- 问题项；
- 已找到；
- 未找到；
- 重复；
- 条件异常。

“问题项”包含未找到、重复和条件异常。

如果存在问题项，搜索完成后默认打开“问题项”；如果全部唯一匹配，则默认显示“全部”。筛选只改变显示，不改变底层搜索结果。

### 8.3 结果表格

表格列为：

| 列 | 内容 |
|---|---|
| 序号 | 原始条件序号 |
| 状态 | 已找到、未找到、重复或条件异常 |
| 分类 | 条件使用的分类显示名，缺失时显示明确占位 |
| 属性名 | 条件使用的属性显示名 |
| 匹配方式 | equals 或 contains |
| 查询值 | 原始查询值 |
| 匹配数 | 正常执行后的对象数量 |
| 说明 | 当前范围内的结果解释或异常原因 |

默认保持原始条件顺序。用户筛选问题项后，问题项仍按原始条件顺序排列，便于与输入 XML 对照。

### 8.4 状态样式

- 已找到：绿色状态文字，普通白色行；
- 未找到：浅红背景，红色状态文字；
- 重复：浅橙背景，橙红状态文字，并显示实际匹配数量；
- 条件异常：浅黄色背景，深黄色状态文字和具体原因；
- 选中行仍必须清晰可辨，状态色不能覆盖系统选中反馈。

### 8.5 表格交互

- 双击已找到、未找到或条件异常行：切换到搜索条件页，并选中对应原始条件；
- 双击重复行：把该条件命中的全部重复对象写入 Navisworks 当前选择，并保持结果页可见；
- 重复对象选择只用于定位，不触发隐藏；
- 长文本通过列宽、自动省略和悬停提示保证可读性；
- 表格为只读，条件修改仍在搜索条件页完成。

## 9. 结果失效规则

以下操作发生后，现有搜索结果立即失效并从结果页清除：

- 导入新的 XML；
- 添加条件；
- 编辑条件；
- 删除条件；
- 清空条件。

结果失效后：

- 清空顶部统计和结果表格；
- 禁用导出结果和创建选择集；
- 结果页显示“条件已变化，请重新执行搜索”；
- 不允许使用旧结果执行任何模型状态操作。

## 10. 隐藏安全门禁

### 10.1 放行条件

只有同时满足以下条件，才允许进入当前的隐藏确认流程：

1. 至少存在一条搜索条件；
2. 所有条件都正常执行；
3. 每条条件都恰好匹配 1 个对象；
4. 最终保留集合非空；
5. 写入 Navisworks 后实际选择非空；
6. STR 保护规则通过现有流程处理；
7. 用户在隐藏确认弹窗中明确选择继续。

### 10.2 阻断行为

只要存在未找到、重复或条件异常：

- 不调用隐藏服务；
- 自动切换到结果页并显示“问题项”；
- 显示各问题状态数量；
- 保留结果供查看、导出和定位；
- 不提供绕过门禁的继续隐藏选项。

这是严格约束，不是普通警告。

### 10.3 其他操作

- 仅查找模式仍可展示全部结果和选中已匹配对象；
- 导出结果保持可用；
- 如果存在已匹配对象，创建选择集可以继续用于人工核查；
- 创建选择集不代表结果通过唯一性验证，也不能绕过隐藏门禁。

## 11. 数据流

1. 用户执行搜索。
2. 插件快照条件列表和当前选择范围。
3. 匹配引擎逐条验证并执行条件。
4. 每条条件生成包含快照、状态、说明和匹配对象的结果。
5. 搜索编排汇总状态和对象集合。
6. 结果页展示统计，并根据是否存在问题项设置默认筛选。
7. 如果存在问题项，隐藏流程被阻断。
8. 如果全部唯一匹配，继续现有 STR 保护、选择写入和隐藏确认。
9. 导出与日志使用同一份结果数据。

## 12. 导出设计

CSV 至少包含以下字段：

- 条件序号；
- 状态；
- 分类；
- 属性名；
- 匹配方式；
- 查询值；
- 匹配数量；
- 状态说明；
- 匹配对象详情。

重复结果的对象详情应能区分多个对象，至少保留显示名称和实例标识。未找到和条件异常行也必须导出，不能只导出命中对象。

## 13. 日志设计

普通结果日志和诊断日志需要增加：

- 四种状态的汇总数量；
- 每条条件的完整快照；
- 每条条件的状态和说明；
- 重复条件的匹配对象数量；
- 隐藏门禁是否通过；
- 阻断隐藏的具体状态数量。

日志写入失败继续沿用非致命处理，不影响已经完成的模型搜索。

## 14. 错误处理

| 场景 | 行为 |
|---|---|
| XML 格式损坏或缺少必要结构 | 中止导入并提示，不生成搜索结果 |
| 条件信息不完整 | 标记条件异常，继续处理其他条件 |
| 分类发现失败 | 标记条件异常，不伪装成未找到 |
| 条件正常但零命中 | 标记未找到 |
| 条件命中多个对象 | 标记重复并保留全部对象引用 |
| Search API 抛出异常 | 中止本次搜索，不展示部分结果 |
| 重复对象定位失败 | 显示定位错误，不改变结果状态 |
| 导出失败 | 显示错误，不清除当前结果 |

## 15. 兼容性与性能

- 继续使用 Navisworks 原生 Search API；
- 不增加全模型复查；
- 不重新遍历匹配对象以判断重复，直接使用每条结果已有数量；
- 结果筛选只作用于内存数据，不重复执行搜索；
- 继续使用 .NET Framework 4.8 和现有 WinForms 技术栈；
- 不改变插件清单、入口和部署目录。

## 16. 验收标准

### 16.1 状态判定

- 条件命中 0 个对象时显示未找到；
- 条件命中 1 个对象时显示已找到；
- 条件命中 2 个或更多对象时显示重复；
- 分类无法识别时显示条件异常；
- 四种状态统计与表格行数一致。

### 16.2 可视化与交互

- 搜索结束后自动进入结果页；
- 存在问题时默认显示问题项；
- 表格展示完整条件和实际匹配数；
- 相同查询值的不同条件可以区分；
- 双击未找到或条件异常行能定位原条件；
- 双击重复行能选中该条件的全部重复对象；
- 不同 DPI 下没有标题、按钮或单元格文字裁切。

### 16.3 安全

- 存在任意未找到时不能隐藏；
- 存在任意重复时不能隐藏；
- 存在任意条件异常时不能隐藏；
- 全部条件唯一匹配时，现有隐藏确认流程正常；
- 零匹配和空实际选择的原有保护继续有效；
- STR 保护逻辑保持不变。

### 16.4 结果生命周期

- 添加、编辑、删除、清空或重新导入条件后，旧结果立即失效；
- 结果失效后不能导出旧结果、创建旧选择集或执行模型状态操作；
- 再次搜索后生成新的完整快照。

### 16.5 导出与日志

- CSV 包含完整条件、四态状态、匹配数和说明；
- 重复行导出多个对象的可区分信息；
- 日志准确记录隐藏被放行或阻断的原因。

## 17. 实施边界

预计修改范围：

- 搜索结果数据模型；
- 匹配引擎的结果构造；
- 主搜索流程的状态汇总和隐藏门禁；
- 结果页布局、筛选和行交互；
- 条件变更后的结果失效处理；
- CSV 导出；
- 普通日志和诊断日志；
- README、CHANGELOG 和 Obsidian 项目知识文档。

实现时不进行与本功能无关的重构。主窗口职责较多的问题可以记录为后续技术债，但本次只提取直接服务于结果状态和筛选的最小辅助逻辑。
